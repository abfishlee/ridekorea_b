# 8. 네이버 지도 최소 비용 전략 (Naver Map Cost-Minimization)

> 네이버를 쓰되 **과금 지점을 구조적으로 최소화**한다. 원칙: **네이버엔 지도 렌더만, 검색·좌표변환·라우팅은 무료 대안(공공데이터/자체)으로.**

## 과금 구조 핵심 (먼저 이해)
- **종량(구간) 요금제** — 호출/뷰 생성 건당 과금.
- **무료 이용량은 대표 계정에 한해** Web Dynamic Map / Static Map / Geocoding / Reverse Geocoding 에만 제공.
- **Mobile Dynamic Map(네이티브 SDK)과 Directions(길찾기)는 무료 이용량 없음** → 첫 호출부터 과금.
- 건수 산정: Mobile Dynamic Map = 지도 뷰 생성(onCreate) 1회당 1건 / Web Dynamic Map = 로딩 시에만 1건(zoom·marker 등 인터랙션 미과금).
- 정확한 무료량/단가는 NCP 콘솔·요금계산기에서 확인(2023.1 개정 이력 있음).

## 본 프로젝트의 렌더러 분배 (비용/성능 균형)
| 화면 | 렌더러 | 이유 |
|---|---|---|
| **Ride(주행)** | 네이티브 SDK(Mobile Dynamic Map) | 실시간 궤적·마커 성능 필수. 과금 감수하되 §아래로 최소화 |
| **Explore 피드 썸네일** | **Static Map** | 무료 이용량 대상, 정적이라 충분 |
| **Explore 루트 상세 / Diary** | **Web Dynamic Map(WebView)** | 무료 이용량 대상, 상호작용 가벼움 |

→ 가장 비싼 네이티브 SDK는 **주행 화면 한 곳**에만 쓰고, 나머지는 무료 구간 대상 렌더러로 돌린다.

## 체크리스트 (개발 규칙)
1. **검색/좌표변환은 네이버 유료 API를 호출하지 않는다.** POI(식당·수리소·숙박·인증센터)는 `pois`/공공데이터에 **좌표 포함 형태로 사전 적재** → 런타임 Geocoding 0회. 자체 검색은 Supabase(`pois`/`routes`) 텍스트 검색으로.
2. **라우팅은 네이버 Directions 호출 0건.** MVP는 저장된 `planned_geom` 폴리라인을 따라가고 이탈 거리만 계산(엔진 불필요). 일반도로 자전거 라우팅은 v2 자체 OSM 엔진.
3. **지도 인스턴스 캐싱.** 화면 재진입 시 지도/WebView를 언마운트·재생성하지 않고 보존(특히 Web Dynamic Map은 "로딩 시 1건"이라 인스턴스 보존이 곧 절감). 탭 전환 시 Ride 지도는 파괴하지 말고 숨김 처리.
4. **Ride 화면 진입 최소화.** 네이티브 SDK 과금은 뷰 생성당 1건이므로, 불필요한 Ride 화면 마운트/리마운트를 막는다(라이딩 시작 시 1회 생성·유지).
5. **콘솔 일/월 이용 상한선 설정** → 비정상 과금·폭주 차단.
6. **클라이언트 ID는 환경변수(EAS Secrets / app config)로 주입** + 도메인/번들ID 제한. 평문 하드코딩 금지.
7. **정적 화면은 Static Map 우선.** 공유 카드·미리보기·썸네일은 Static Map(무료 대상).
8. **사용량 모니터링.** NCP 콘솔 Usage Statistics로 월별 추세 추적, 무료 한도 근접 시 점검.

## 효과
과금 트리거가 사실상 **"Ride 화면을 몇 번 새로 띄우느냐"** 하나로 수렴한다. Explore/Diary는 무료 이용량 안에서 거의 무료, 검색·좌표변환·라우팅 비용은 0. 초기·파일럿은 사실상 무료~소액으로 운영 가능.

## 비고 (지도 제공자 교체 대비)
구글의 한국 턴바이턴이 정식 출시되면 다국어+라우팅이 강해질 수 있으므로, 지도 호출은 **얇은 래퍼(provider 어댑터)**로 감싸 교체 가능하게 둔다. (네이버 native ↔ Web ↔ Static ↔ 향후 Google)

## 비용 충당
사업화자금(관광벤처·예비창업패키지) 집행 시 지도 API·클라우드 비용은 '지급수수료' 비목으로 **협약기간 내** 집행 가능성이 높음(주관기관 PD 사전확인). 협약 종료 후 운영비는 지속 모델(지자체 B2G·제휴·후속펀딩)로 충당. (상세: `7_mvp_scope_and_roadmap.md`)
