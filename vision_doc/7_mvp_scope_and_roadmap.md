# 7. MVP 범위 · 로드맵 · 리스크 (Scope, Roadmap & Risks)

## 🪜 단계별 범위

### MVP (출시 가능한 최소 — 한 코스/한 지역에 집중)
- 소셜 로그인(구글/애플) + 프로필 자동 생성
- **Explore**: 공식 루트(시드) + 공개 유저 루트 피드, 루트 상세(지도+스팟), `[가져오기]`
- **Ride**: 네이티브 지도, 실시간 트랙(Blue), 이탈 표시(Pink), 사진핀, **오프라인 아웃박스 동기화**
- **Diary**: 내 라이딩 타임라인, 공개 발행(Publish)
- **Wallet**: 바우처 claim/목록, (간이) 스탬프
- 파일럿: **서울–부산(4대강)** 1개 코스 + **1개 지역** 바우처 + 핵심 POI(수리소·식당·숙박) 시드

### v1
- 물류 가이드 게시판(`logistics_tips`), 좋아요/댓글, 종주 스탬프 패스포트
- 다국어(EN/JP) i18n, 메뉴 번역, POI 다국어(TourAPI)
- FastAPI 월간 ETL 자동화(`pois` upsert)

### v2+
- 일반도로 자전거 라우팅(자체 OSM/GraphHopper), 친구 실시간 궤적(Realtime)
- AI 추천(코스·명소), 매장 셀프서비스 바우처 등록 + 정산, 매장용 redeem 앱/웹
- 지자체 B2G 성과 대시보드(유입·체류·소비)

### MVP 비목표(명시적 제외)
턴바이턴 내비, 친구 실시간 위치, 인앱 결제/정산, 라우팅 엔진, AI 추천.

## 🌱 콜드스타트 시딩 전략 (가장 중요)
커뮤니티가 비어 있으면 앱의 핵심 가치가 0이다. **출시 전에 채운다.**
1. 서울–부산 코스에 **양질의 시드 기행문 10~20개**를 운영진/섭외 라이더 명의로 사전 제작(사진+스팟+메모 포함).
2. 기존 외국인 라이더 커뮤니티·블로그·GPX를 (동의 하에) 임포트·재구성.
3. 물류 가이드(인천공항→4대강 점프 등) 5~10건 사전 작성.
4. 커뮤니티가 차기 전에도 **공식 루트 + 오프라인 지도 + 번역**만으로 단독 가치를 갖도록 보장(빈 화면 금지).

## 🗂️ 기본 제공 데이터 소스 (FastAPI ETL 대상)
월 1회 수집 → 정규화 → `pois`/`certification_centers`/공식 `routes` upsert. (출처·갱신일 메타 저장)
- **도로/종주길/스탬프(월간 갱신 핵심)**: 전국자전거도로표준데이터, 행정안전부 자전거길 DB, 국토종주 인증센터 위치.
- **숙박·음식·관광(다국어)**: 한국관광공사 TourAPI(영/일/중 지원).
- **캠핑**: 고캠핑(GoCamping) API.
- **자전거 코스/주변정보**: 두루누비 코스 API.
- **보조**: 공공자전거 대여소, 기상청 단기예보, OSM(수리·렌탈샵).
> 라이선스: 데이터셋별 공공누리 유형·출처표기 의무를 `pois.meta`에 함께 저장.

## ⚠️ 리스크 & 대응

| 리스크 | 영향 | 대응 |
|---|---|---|
| **콜드스타트(콘텐츠 공백)** | 핵심 가치 0 | 위 시딩 전략, 좁은 출시(1코스) |
| **양면시장(바우처 매장↔라이더)** | 바우처 미작동 | 1개 지역·소수 매장부터, 지자체 제휴로 한쪽 보조 |
| **금전성 바우처 법적/정산** | 출시 차단 가능 | MVP는 "매장 제시형 할인쿠폰"로 시작, 전자금융/상품권 규제·정산은 법무 검토 후 v2 |
| **지도 SDK 비용(네이티브)** | 운영비 부담 | Ride만 네이티브, 나머지 Web/Static, 콘솔 상한선 |
| **백그라운드 위치(배터리·권한·심사)** | 추적 실패/리젝 | Dev Client/EAS, 목적 고지 문구, throttle |
| **niche TAM(외국인 종단)** | 상업 스케일 한계 | 공익·관광 펀딩 모델(관광벤처/예비창업/지자체 B2G) |
| **외국인 개인정보(GDPR 등)** | 법적 리스크 | 위치·사진·소셜 처리방침, 최소 수집, 권한 목적 고지 |
| **공개 콘텐츠 모더레이션** | 품질·안전 | `reports` 신고 + 운영 검수 큐 |
| **부정 바우처(위치 스푸핑/파밍)** | BM 훼손 | 서버 위치 재검증·unique·수량·레이트리밋(문서6 §3) |

## 📊 성공 지표
- 활성화: 가입→첫 `가져오기` 전환율, 첫 라이딩 시작률
- 핵심 루프: 라이딩 완주율, 라이딩당 평균 스팟 수, 오프라인 동기화 성공률
- 콘텐츠 플라이휠: 공개 기행문 발행 수, 재방문 생산자 수, 좋아요/댓글
- BM/공익: 바우처 claim→redeem 전환율, 평균 체류일(지역 소비 대용)

5. Diary 타임라인 → 발행
6. regions/vouchers 시드 + claim_voucher Edge Function + Wallet
7. FastAPI ETL(파일럿 POI) → 월간 스케줄
