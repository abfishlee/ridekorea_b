# 1. 개발 목표 및 가이드 (Development Goals & Guidelines)

> 이 문서군의 북극성(North Star)은 `0_sasaki_scenario.md`다. 모든 기능 결정은 "사사키의 여정을 더 좋게 만드는가?"로 검증한다.

## 🎯 서비스의 궁극적 목표 (Core Vision)
RideKorea는 단순한 자전거 속도/경로 추적 앱(예: Strava)이 아니다. **"대한민국을 방문한 외국인 자전거 여행객(인바운드)이 길 잃을 두려움 없이 로컬 문화를 탐험하고, 그들의 모든 여정을 한 편의 감성적인 기행문으로 완성시켜 주는 소셜 여행 플랫폼"**이다.

## 🚀 3대 핵심 개발 철학
1. **기록이 아닌 '기행문(Diary)'**: 위치·속도 같은 차가운 데이터에 더해 사진·감정·머문 시간 같은 따뜻한 맥락(Context)을 함께 묶어 저장한다.
2. **실수(경로 이탈)의 긍정적 치환**: 이탈 시 경고음 대신, 원래 경로(Aero Blue)와 이탈 경로(Adventure Pink)를 시각적으로 분리해 "나만의 개척로"로 포장한다. (알고리즘: `6_core_flows_and_algorithms.md` §2)
3. **지역 경제(로컬)와의 상생**: 특정 지역 진입 시 위치 기반 지역 상권 바우처를 제공하는 것이 1차 BM이자 공익 명분이다. (단, 금전성 바우처의 법적·정산·부정사용 리스크는 `7_mvp_scope_and_roadmap.md` §리스크 참조)

## 🧭 MVP 원칙 (Scope Discipline)
"완성형 시나리오"를 한 번에 만들지 않는다. MVP는 **한 코스(서울–부산 4대강)**와 **한 파일럿 지역 바우처**에 집중한다. 상세 범위는 `7_mvp_scope_and_roadmap.md`.

- **MVP에 반드시 포함**: 소셜 로그인 → Explore 피드(공식 루트 + 공개 유저 루트) → 루트 가져오기 → Ride(실시간 궤적·사진핀·이탈 표시·오프라인 큐) → Diary(내 라이딩 타임라인) → Wallet(바우처 claim/목록).
- **MVP 비목표(Non-Goals)**: 턴바이턴 내비게이션, 친구 실시간 위치 공유, 인앱 결제/정산, 일반도로 자전거 라우팅 엔진, AI 추천.

## ⚠️ 콜드스타트 원칙
커뮤니티 콘텐츠가 이 앱의 심장이다. **출시 첫날 "빈 앱"을 절대 보여주지 않는다.** 서울–부산 코스에 양질의 시드 기행문 10~20개를 사전 적재한다(운영진·인플루언서·기존 외국인 라이더 섭외). 커뮤니티가 차기 전에도 공식 루트 + 오프라인 지도 + 번역만으로 가치가 있게 설계한다.

## 💻 AI 협업 개발 가이드라인 (For GPT / Claude)
* **Component-Driven**: 모든 UI는 재사용 가능한 작은 컴포넌트로 분리.
* **Strict Typing**: TypeScript로 모든 Props/State와 **Supabase DB 스키마 타입을 자동 생성**해 사용한다(`supabase gen types typescript`). 손으로 타입을 적지 말 것.
* **Offline-First (Outbox 패턴)**: GPS 좌표·사진·메모는 기기(SQLite) **아웃박스 큐**에 먼저 쓰고, 네트워크 복구 시 백그라운드로 Supabase에 동기화한다. 모든 로컬 레코드는 **클라이언트 생성 UUID**를 PK로 써서 재전송이 멱등(idempotent)하도록 한다. (사진 "즉시 업로드" 금지 — 음영지역에서 실패한다)
* **Server logic는 Edge Function**: 트랜잭션·검증이 필요한 로직(바우처 발급, 루트 가져오기, 좋아요 토글, 라이딩 종료 집계)은 클라이언트가 아니라 Supabase **Edge Function**(service role)에서 처리한다. 클라이언트의 anon key를 신뢰하지 않는다.
* **보안 기본값**: 모든 public 테이블에 **RLS(Row Level Security)**를 켠다. RLS 없는 테이블 생성 금지. (정책: `5_database_schema_sql.md`)

## 📊 성공 지표 (North-Star Metrics)
- 활성화: 가입 → 첫 루트 "가져오기" 전환율
- 핵심 루프: 라이딩 완주율, 라이딩당 평균 스팟(사진핀) 수
- 콘텐츠 플라이휠: 공개 기행문 발행 수, 재방문 콘텐츠 생산자 수
- BM/공익: 바우처 claim → 실제 사용(redeem) 전환율, 평균 체류일(지역 소비 대용 지표)
